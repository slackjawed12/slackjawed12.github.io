[Design Pattern] 상태 패턴(State Pattern) 적용

August 5, 2023

5min read

Overview

    상태 패턴은 보통 어떤 객체의 행위를 상태에 따라 다르게 하도록 구현해야 하는 경우 적용되는 디자인 패턴입니다. 게임에서 AI를 간단히 구현할 때 종종 사용한다고 합니다. 상태패턴을 적용한 간단한 예시로, 백준 1913번 문제를 풀어보겠습니다.

1. 상태패턴 없이 구현

    홀수 N이 주어질 때, 가로 N, 세로 N 크기의 표를 달팽이 모양으로 N×NN \times NN×N부터 1까지 채우는 문제입니다. 아래로 움직이는 중에 방향을 바꿔야 하는 조건에 해당하면 오른쪽으로 움직이게 바꾸고, 오른쪽으로 움직이던 중에 방향을 바꿔야 하면 위로 움직이도록 구현하면 됩니다. 이런 방향 변화를 간단히 나타내면, DOWN -> RIGHT -> UP -> LEFT -> DOWN 순서입니다. 길이가 4인 배열에 modular 연산을 적용하면 쉽게 구현할 수 있습니다.

    의사코드로 생각해보면, setDirection 함수는 범위를 벗어났거나 다음 자리에 이미 값이 들어가있을 때 (i+1)%4를 반환하고, 그런 경우가 아니면 현재 인덱스를 반환하면 됩니다. 구현 자체가 명확하고, 코드도 깔끔한 편힙니다. 다만 이 문제의 맥락이 비즈니스 로직의 일부를 작성하는 것이라면 확장에 유연해야 합니다.

2. 상태패턴을 적용한 풀이

움직이는 객체를 나타내는 클래스를 Agent라고 정의하겠습니다. 위치와 num, 그리고 움직이는 상태인 state를 갖고있습니다.

이제 State들을 정의해보겠습니다. MoveState는 handle 메서드가 있는 interface로 정의하고, MoveState의 구현체들은 handle 메서드를 재정의합니다. handle 메서드는 각 state에서의 동작과 다음 상태를 반환합니다. 예를 들어, 왼쪽으로 움직이는 상태를 나타내는 LeftState 객체는 다음처럼 구현할 수 있습니다.

LeftState처럼 RightState, UpState, DownState를 구현하면 됩니다. 이제 Agent 객체를 초기화하고, 아래 루프에 따라 움직이게 하면 됩니다.
